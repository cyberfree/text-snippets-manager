## 🐛 Il Problema (v2.5):

```
User flow v2.5:
1. User click "Incolla" nel popup
2. popup.js: sendMessage('pasteSnippet')
3. background.js: executeScript(pasteText)  ← IMMEDIATO
4. pasteText: clipboard.write()
5. ❌ Browser: "Document is not focused"
   └─> Popup ANCORA APERTO, ha ancora focus
6. popup.js: window.close()  ← TROPPO TARDI

Risultato: Clipboard API fallisce sempre dal popup
```

---

## ✅ La Soluzione (v2.6):

### **Chiudi popup PRIMA, incolla DOPO**

```javascript
// popup.js
async function pasteSnippet(id) {
  const snippet = snippets[id];
  const textToPaste = snippet.text;
  
  // ✅ 1. CHIUDI POPUP IMMEDIATAMENTE
  window.close();
  
  // ✅ 2. POI manda messaggio
  await chrome.runtime.sendMessage({ 
    action: 'pasteSnippet', 
    text: textToPaste 
  });
}

// background.js
} else if (request.action === 'pasteSnippet') {
  // ✅ 3. ASPETTA che popup si chiuda
  setTimeout(() => {
    // Ora page ha focus
    chrome.scripting.executeScript({
      target: { tabId: tabs[0].id },
      func: pasteText,
      args: [request.text]
    });
  }, 100);  // ← 100ms per chiusura completa
}
```

**Timeline corretta**:
```
t=0ms    : User click "Incolla"
t=1ms    : window.close() chiamato
t=10ms   : Popup inizia chiusura
t=50ms   : Browser restituisce focus a page
t=100ms  : executeScript inietta pasteText
t=110ms  : pasteText: clipboard.write()
t=111ms  : ✅ SUCCESS - page ha focus
```

## ⚙️ Dettagli Tecnici:

### Delay di 100ms:

**Perché serve?**
- Popup chiusura non è istantanea
- Browser needs time to:
  1. Destroy popup DOM
  2. Release popup focus
  3. Restore focus to page
  4. Update document.activeElement

**È abbastanza?**
- ✅ Su macchine normali: Sì
- ⚠️ Su macchine lente: Potrebbe servire 150ms
- 🐛 Se fallisce: Aumenta delay a 200ms (dimmi)

### Perché non più toast?

```javascript
async function pasteSnippet(id) {
  window.close();  // ← Popup DOM distrutto qui
  
  showToast('...');  // ← Impossibile, popup non esiste più
}
```

**Alternative**:
1. Toast nel page content (complicato)
2. Notification browser (troppo invasivo)
3. Solo highlight verde (soluzione scelta)

## 💡 Alternative Considerate (e scartate):

### Opzione A: Focus stealing API
```javascript
chrome.windows.update(windowId, { focused: true });
```
❌ Requires `windows` permission (troppo invasivo)

### Opzione B: Copy-only mode
```javascript
clipboard.writeText(text);
showToast("Premi Ctrl+V");
```
❌ UX peggiore, richiede azione manuale

### Opzione C: Zero delay
```javascript
window.close();
// no setTimeout
executeScript(...);
```
❌ Race condition, fallisce 50% delle volte

**Opzione D: window.close() + 100ms delay** ✅
- Semplice
- Affidabile
- Minimo trade-off UX
